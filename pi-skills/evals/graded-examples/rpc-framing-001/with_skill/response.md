Per `pi-mono/packages/coding-agent/docs/rpc.md`, RPC mode uses strict JSONL with LF (`\n`) as the only record delimiter between messages. Node `readline` is not protocol-compliant for this framing because it also treats **U+2028** (LINE SEPARATOR) and **U+2029** (PARAGRAPH SEPARATOR) as line breaks. Those code points can appear inside JSON string values, so a readline-based splitter could incorrectly break a single JSON object across lines.
