In `pi-mono/packages/coding-agent/src/core/resource-loader.ts`, **`extendResources`** records extension skill paths, then **`mergePaths`** merges the previous **`lastSkillPaths`** with the new paths (extension paths come **after** the primary list). **`mergePaths`** walks **`[...primary, ...additional]`** and **keeps the first resolved path** when duplicates appear, so ordering from the earlier **`reload()`** pass is preserved for the same filesystem location. Skill **name** collisions are still resolved later by **`loadSkills`** (first registered name wins), but the path merge/dedup behavior for **`extendResources`** is entirely in this file’s **`mergePaths`** / **`extendResources`** flow.
