Per `pi-mono/packages/coding-agent/docs/rpc.md`, if you are building a Node.js application you should prefer using **`AgentSession`** from `@mariozechner/pi-coding-agent` instead of spawning a subprocess for RPC. The document points TypeScript readers at the concrete API in `pi-mono/packages/coding-agent/src/core/agent-session.ts` (relative link `src/core/agent-session.ts` in the doc).
