According to `pi-mono/packages/coding-agent/docs/compaction.md`, pi auto-compacts when the context token count exceeds the model's limit minus the `reserveTokens` budget. The cut point is chosen so that the most recent `keepRecentTokens` worth of messages are preserved, and the algorithm avoids splitting tool results (a tool call and its result stay together). You can tune both `reserveTokens` and `keepRecentTokens` in `settings.json` under the `compaction` key. Chained compactions link via `firstKeptEntryId`. You can also disable auto-compaction in settings while keeping manual `/compact` available.
