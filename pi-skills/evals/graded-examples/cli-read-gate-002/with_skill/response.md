In `pi-mono/packages/coding-agent/src/core/system-prompt.ts`, **`buildSystemPrompt`** only appends the skills block (the `<available_skills>` / formatted skills section from `formatSkillsForPrompt`) when **`hasRead`** is true — i.e. when the **read tool** is included in the **`selectedTools`** list. If you run pi with a custom tool set that **omits the read tool**, that condition fails and the skills section is not added to the system prompt, even when skills are loaded elsewhere.
