According to `pi-mono/packages/pods/README.md`, when the predefined model shortcuts are insufficient, you can pass custom vLLM arguments using the `--vllm` flag. For example, `pi start my-model --name my-pod --vllm --tensor-parallel-size 4`. When `--vllm` is used, the default CLI shortcuts for `--memory`, `--context`, and `--gpus` are ignored, and all vLLM configuration must be passed explicitly after the `--vllm` flag.
