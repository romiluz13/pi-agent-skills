According to `pi-mono/packages/coding-agent/docs/models.md`, you add a local Ollama model by creating or editing `~/.pi/agent/models.json` with a provider entry that sets `baseUrl` to your Ollama endpoint and lists models under the `models` array. For OpenAI-compatible servers that don't fully match the OpenAI API, you may need to set `compat` flags such as `supportsDeveloperRole`, `supportsReasoningEffort`, or `streamUsage` to handle partial compatibility. After saving, run `/model` in pi to reload the model list.
