According to `pi-mono/packages/coding-agent/docs/json.md`, `--mode json` emits a stream of JSON line events to stdout including `agent_start`, `agent_end`, `turn_start`, `turn_end`, `message_start`, `message_update`, `message_end`, `tool_execution_start`, `tool_execution_end`, and others like `compaction_start`/`compaction_end`. Unlike `--mode rpc` which is bidirectional (stdin commands + stdout responses), `--mode json` is read-only observation — you cannot send commands back. You can filter events with jq, for example: `pi --mode json -p "hello" | jq 'select(.type == "message_update")'`.
